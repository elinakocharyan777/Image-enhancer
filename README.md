# ML-модель для улучшения изображений

Проект выполнен в рамках онлайн-практики VK Education

## Описание проекта

Проект представляет собой браузерную систему для улучшения изображений в реальном времени. Система принимает изображение от пользователя, анализирует его параметры и автоматически применяет коррекцию яркости, контрастности и цветности.

Обработка выполняется на стороне клиента, без отправки изображения на сервер. Для асинхронной обработки используется Web Worker, что позволяет не блокировать интерфейс браузера во время выполнения задачи

## Цель проекта

Разработать систему, реализующую улучшение изображений в реальном времени посредством запуска ML-модели или алгоритма подбора параметров коррекции в браузере пользователя

## Поддерживаемые форматы

На вход система принимает изображения следующих форматов:

- JPG / JPEG
- PNG
- BMP
- HEIC / HEIF

Формат HEIC / HEIF предварительно конвертируется в JPEG на стороне клиента, так как браузеры не всегда поддерживают прямое декодирование HEIC через Canvas API

Готовое изображение выгружается в формате PNG

## Основные возможности

- загрузка изображения пользователем
- предпросмотр исходного изображения
- автоматический анализ изображения
- подбор параметров коррекции яркости, контрастности и цветности
- асинхронная обработка изображения через Web Worker
- отображение прогресса выполнения задачи
- возможность прерывания обработки
- отображение информации о задаче
- визуальное сравнение исходного и обработанного изображения
- скачивание готового результата

## Логика работы системы

1. Пользователь выбирает изображение
2. Система проверяет формат файла
3. Если выбран HEIC / HEIF, файл конвертируется в JPEG
4. Изображение отображается в интерфейсе
5. Пользователь запускает обработку
6. Создаётся задача обработки и формируется `taskId`
7. Изображение загружается в Canvas
8. Пиксели изображения передаются в Web Worker
9. Worker анализирует изображение и подбирает параметры коррекции.
10. Worker применяет коррекцию яркости, контрастности и цветности.
11. Интерфейс получает прогресс обработки.
12. После завершения пользователь видит результат и может скачать его в формате PNG.

## API модуля обработки изображений

В проекте реализован отдельный API-модуль `ImageEnhancerAPI`.

### `createTask(file)`

Создаёт задачу обработки изображения и возвращает идентификатор задачи.

### `getStatus(taskId)`

Возвращает текущий статус задачи, прогресс выполнения и информацию об ошибке, если она возникла.

### `cancelTask(taskId)`

Прерывает выполнение задачи обработки изображения.

### `getResult(taskId)`

Возвращает ссылку на готовое обработанное изображение.

### Событие `statuschange`

Возникает при изменении статуса или прогресса выполнения задачи.

### Событие `result`

Возникает после успешного завершения обработки изображения.

### Событие `cancelled`

Возникает после прерывания задачи.

### Событие `error`

Возникает при ошибке обработки.

## Алгоритм улучшения изображения

Система анализирует изображение и рассчитывает несколько признаков:

- среднюю яркость
- контрастность
- среднюю насыщенность

На основе этих признаков подбираются параметры коррекции:

- значение изменения яркости
- коэффициент контрастности
- коэффициент насыщенности

После этого выбранные параметры применяются к каждому пикселю изображения.

## Ограничения

В проекте учтены следующие ограничения:

- обработка изображений до 15 Мп
- максимальное время обработки- до 30 секунд
- асинхронная работа без блокировки браузера
- поддержка массовых современных браузеров
- выгрузка результата в формате PNG

## Как запустить проект

Проект использует ES-модули, поэтому его нужно запускать через локальный сервер.

В папке проекта выполнить команду:

```bash
python3 -m http.server 8000
```

После этого открыть в браузере:

```text
http://localhost:8000
```
